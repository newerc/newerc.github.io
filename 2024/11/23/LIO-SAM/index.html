LIO-SAM：通过平滑和映射的紧耦合激光雷达惯性里程计

概述

  • 实现高精度、实时的移动机器人轨迹估计和地图构建

  • 在因子图上制定激光雷达惯性里程计，允许将多种相对和绝对测量（包括闭环）(从不同来源）作为因子合并到系统中

  • 惯性测量单元 (IMU) 预积分的估计运动可消除点云的倾斜，并为激光雷达里程计优化生成初始猜测。获得的激光雷达里程计解用于估计IMU的偏差。

  • 确保实时的高性能
    • 将旧的激光雷达扫描边缘化以及进行姿态优化（❌将激光雷达扫描与全局地图进行匹配）
    • 局部尺度扫描匹配
    • 选择性引入关键帧以及将新关键帧注册到固定大小的先前“子帧”集的有效滑动窗口

介绍

  • 智能移动机器人成功的基本先决条件：状态估计、定位、建图

  • 基于视觉和激光雷达传感技术→SLAM→支持移动机器人六自由度状态估计

    • 视觉：单目/立体相机对连续图像特征进行三角测量→确定相机运动
      • 优点：适合地点识别
      • 缺点：对初始化、照明和范围的敏感性使得它们在单独用于支持自主导航系统时并不可靠
    • 激光雷达：很大程度上不受光照变化的影响，更适合直接捕获3D空间中的环境细节；本文重点研究

  • LOAM存在的局限性

    • 通过将其数据保存在全局体素地图中，通常很难执行回环检测并结合其他绝对测量（例如 GPS）进行姿态校正。
    • 当该体素图在特征丰富的环境中变得密集时，其在线优化过程的效率就会降低。
    • 在大规模测试中会出现漂移，因为它的核心是基于扫描匹配的方法。

  • LIO-SAM主要贡献

    • 建立在因子图之上的紧耦合激光雷达惯性里程计框架，适用于多传感器融合和全局优化。
    • 一种高效的、基于局部滑动窗口的扫描-匹配方法，通过将选择性选择的新关键帧注册到一组固定大小的先前子关键帧来实现实时性能。
    • 所提出的框架通过各种规模、车辆和环境的测试进行了广泛的验证。

相关文献参考

  • 激光雷达里程计通常使用ICP和GICP等扫描匹配方法查找两个连续帧之间的相对变换来执行
  • 激光雷达通常与其他传感器（例如IMU和GPS）结合使用，用于状态估计和绘图
    • 分类1：松耦合融合
      • LOAM：引入IMU来消除激光雷达扫描的倾斜，并为扫描匹配提供运动先验，然而IMU不参与算法的优化过程
      • LeGO-LOAM：对IMU测量的融合与LOAM相同
      • 另一种方法：使用拓展卡尔曼滤波器（EKF）
    • 分类2：紧耦合融合：准确性较高（目前研究的主要焦点）
      • R-LINS：以机器人为中心的激光雷达惯性状态估计器。使用错误状态卡尔曼滤波器以紧密耦合的方式递归地纠正机器人的状态估计，由于缺乏其他可用的状态估计传感器，它在长时间导航过程中会出现漂移
      • LIOM（LIO-mapping）：联合优化激光雷达和IMU测量；用于处理所有传感器测量结果，因此无法实现实时性能

通过平滑和映射的紧耦合激光雷达惯性里程计

系统简述

  • 该系统从3D激光雷达、IMU和可选的GPS接收传感器数据。我们试图通过观察这些传感器来估计机器人的状态及其轨迹。这个状态估计问题可以表述为最大后验 (MAP) 问题。我们使用因子图来模拟这个问题，因为与贝叶斯网络相比，它更适合执行推理。
  • 引入四种类型的因子
    • IMU预积分因子
    • 激光雷达里程计因子
      • 新的激光雷达扫描到达时，首先执行特征提取，通过评估局部区域上点的粗糙度来提取边缘和平面特征。具有较大粗糙度值的点被归类为边缘特征。类似的，平面特征按较小的粗糙度进行分类。在i时刻提取的所有特征组成一个激光雷达帧F$_i$。
      • 使用每个激光雷达帧进行计算并向图中添加因子在计算上是很困难的→关键帧选择。
        • 当机器人姿态的变化与之前的状态X$_{i}$相比超过用户定义的阈值时，选择激光雷达帧 F$_{i+1}$作为关键帧，新保存的关键帧F$_{i+1}$与因子图中的新机器人状态节点 X$_{i+1}$相关联。两个关键帧之间的激光雷达帧将被丢弃←通过这种方式添加关键帧不仅可以实现地图密度和内存消耗之间的平衡，而且有助于保持相对稀疏的因子图，适合实时非线性优化
        • 本实验：添加新关键帧的位置和旋转变化阈值选择为1m和10°
        • 向因子图中添加一个新的状态节点 X$_{i+1}$。与此状态关联的激光雷达关键帧是 F$_{i+1}$。激光雷达里程计因子的生成按以下步骤描述：
          1. 体素图的子关键帧
          2. 扫描匹配
    • GPS因子
      • 解决系统在长时间导航任务中仍出现漂移
      • 引入提供绝对测量的传感器（高度计、指南针和GPS）以消除漂移
      • 收到GPS测量结果→转换为本地笛卡尔坐标系→向因子图中添加新节点以后，将新的GPS因子与该节点相关联
      • 如果 GPS 信号与激光雷达帧不硬件同步，我们会根据激光雷达帧的时间戳在 GPS 测量值之间进行线性插值。
      • 当 GPS 接收可用时，没有必要不断添加 GPS 因子，因为激光雷达惯性里程计的漂移增长非常缓慢。只有当估计的位置协方差大于接收到的 GPS 位置协方差时，我们才添加 GPS 因子
    • 环路闭合因子
      • 一种朴素但有效的基于欧几里德距离的闭环检测方法
      • 本文提出的框架与其他闭环检测方法兼容
      • 作者发现当 GPS 是唯一可用的绝对传感器时，添加闭环因子对于纠正机器人高度的漂移特别有用。这是因为 GPS 的海拔测量非常不准确 - 在我们的测试中，在没有环路闭合的情况下，海拔高度误差接近 100m。
  • 引入用于因子图构建的变量类型
    • 表示机器人在特定时间的状态，归因于图的节点

实验

  • 数据采集：Rotation、Walking、Campus、Park、Amsterdam
  • 测试方法
    • 系统初始化
      • 激光雷达点云通过IMU的预积分去畸变（去除运动扭曲）。
      • 利用初始IMU估计值和扫描匹配进行初始定位。
    • 基准地图与轨迹生成
      • 将激光雷达点云通过因子图优化生成机器人轨迹。
      • 合成地图并与Google Earth等真实数据进行比对。
    • 单独功能验证
      • IMU预积分和激光雷达里程计因素：在没有绝对定位信息的情况下，测试短时间内系统的建图能力和漂移量。
      • GPS因素：通过将GPS位置作为因子图约束，校正长时间运行时的轨迹漂移。
      • 闭环检测：使用欧几里得距离搜索方法检测机器人回到先前位置的可能性，测试闭环校正对漂移的修正效果。
    • 对比方法
      • 与LOAM、LIOM等框架进行建图精度和效率的对比。
      • 在实时和非实时条件下评估系统性能。
  • 实验结果
    • 精度：LIO-SAM生成的地图与真实环境（如Google Earth）对齐良好，轨迹漂移显著减少。
    • 实时性：每帧处理时间短于LOAM和LIOM，可实时运行。
    • 鲁棒性：在复杂环境（如旋转运动、稀疏特征区域）中表现稳定。

结论与讨论

  • 作者提出了 LIO-SAM，这是一种通过平滑和映射实现紧耦合激光雷达惯性里程计的框架，用于在复杂环境中执行实时状态估计和映射
  • 通过在因子图上制定激光雷达惯性里程计，LIO-SAM 特别适合多传感器融合。额外的传感器测量可以作为新因素轻松纳入框架中。提供绝对测量的传感器（例如 GPS、指南针或高度计）可用于消除激光雷达惯性里程计在长时间内或在功能匮乏的环境中累积的漂移。地点识别也可以轻松地集成到系统中。
  • 为了提高系统的实时性能，我们提出了一种滑动窗口方法，可以边缘化旧的激光雷达帧以进行扫描匹配。关键帧有选择地添加到因子图中，并且当生成激光雷达里程计和回环因子时，新关键帧仅注册到固定大小的子关键帧集。这种局部范围而不是全局范围的扫描匹配有利于 LIO-SAM 框架的实时性能。
  • 所提出的方法在跨各种环境的三个平台上收集的数据集上进行了彻底评估。结果表明，与 LOAM 和 LIOM 相比，LIO-SAM 可以获得相似或更好的精度。
  • 未来的工作包括在无人机上测试所提出的系统。
